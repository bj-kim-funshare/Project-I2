**목표(end state):** data-craft 웹을 ① FSD 완전 폐기·`src/components/{atoms,molecules,organisms}` Atomic 완전화, ② `packages/` 모노레포 전부 폐기·단일 `src` 흡수, ③ 모든 데이터 패칭(apiClient ~254곳 + 직접 fetch ~15곳 + axios 1)을 ky 로 완전 전환(@tanstack/react-query 유지), ④ import/export 제외 50줄 초과 모든 코드 behavior-preserving 분해, ⑤ 데드코드 전수 제거, ⑥ 전 파일 다각도 코드리뷰 1회 횡단 — 으로 전환한다. **절대 조건: 기존 모든 기능의 소실·오동작 제로.**

**배경:** 실험 브랜치 `refactor/atomic-monorepo-absorption`(로컬 전용, tip 840b86a3d, 6/11 마지막)은 구조이동·fork-drift parity·일부 클린코드를 이미 완료했으나, 분기(6/10) 이후 i-dev 가 **207커밋** 앞서며 스테일. in-memory 머지 실측: 표면 충돌 36건이나 그중 28건이 file-location 타입 — i-dev 가 분기 후 옛 `packages/` 구조에 **215파일**을 추가/수정했고, 단순 머지 시 삭제된 모노레포 구조가 부활하는 구조적 오염이 진짜 비용. ky 전환은 스테일 브랜치 미수행 신규 작업.

**머지/브랜치 모델:** i-dev 는 멈추지 않는다(다른 세션이 WIP 으로 계속 작업, 회사 성과). 리팩토링은 `refactor/atomic-monorepo-absorption`(되살림 권장) 에서 병행. 평상시 `i-dev → refactor` 주기적 흡수 → 근접 시점에 짧은 i-dev freeze 창에서 최종 흡수·전수 검증 후 `refactor → i-dev` cutover 승격. 흡수 빈도 = 작업 빈도(매 프롬프트가 §공통 흡수 선행). git merge-base 가 흡수 원장이므로 별도 SHA 원장 불필요.

**통합 브랜치 오버라이드(마스터 명시 허가):** plan-enterprise 및 inspection 3종은 통합 브랜치가 i-dev 하드와이어(CLAUDE.md §5·§9, plan-enterprise SKILL.md 의 WIP base·완료 머지 다지점)다. 본 로드맵은 이를 **마스터 명시 허가**로 프롬프트마다 오버라이드한다 — 진입 정렬 검증/WIP base/머지 대상 모두 `refactor/atomic-monorepo-absorption` 으로 대체하고 i-dev HEAD/origin 은 불가침. override 가 다지점 수동 번역이라 누락 1회로 i-dev 가 사일런트 오염될 수 있으므로, **각 프롬프트 §공통 프리앰블에 결정적 완료 단언(C)** 을 적재: 작업 시작/종료 시 i-dev·origin/i-dev·HEAD 3값 불변을 기계적으로 검증하고 위반 시 halt. patch-note 는 중간 프롬프트에서 미발행(plan-enterprise Step 9.2~9.5 생략)하고 cutover 단계에서 일괄 발행한다.

**핵심 제약(전 프롬프트 §공통 프리앰블 고정):** (A) 격리 — 공유 메인 워킹트리(i-dev)는 불가침(checkout/switch 금지). 쓰기 작업은 plan-enterprise 가 override(ii)로 base=refactor 에서 생성한 자기 WIP A worktree(`<wt>`)에서 `git -C <wt>` 로 수행(표준 WIP→머지 모델 유지, base·머지대상=refactor). 읽기전용 inspection(프롬프트 6)은 refactor 브랜치 worktree(예: `../data-craft-worktrees/refactor-atomic`)에서 read. (B) 흡수 — 매 작업 전 `git -C <wt> merge i-dev` + 옛 구조 신규 파일 Atomic 재이주 + 게이트(pnpm typecheck:all && lint && build) + SPOT-CHECK 런타임. (C) 완료 단언 — i-dev/origin/HEAD 3값 불변 기계 검증. inspection 은 B 의 머지/재이주 없이 직전 흡수 상태를 검사만 한다.
